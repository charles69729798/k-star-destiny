## 1. 개요

### 1.1 테스트 목적

본 보고서는 K-Destiny AI 웹서비스의 사용자 인수 테스트(UAT) 결과를 정리한 것이다. 3개 언어(한국어, 영어, 스페인어)에 대해 모든 주요 기능을 테스트하고, 다국어(i18n) 관련 문제점을 발견하여 개선 방안을 제안하는 것을 목적으로 한다.

### 1.2 테스트 범위

| 구분 | 내용 |
|------|------|
| **대상 언어** | 한국어(KO), 영어(EN), 스페인어(ES) |
| **테스트 방법** | Playwright 기반 자동화 브라우저 테스트 |
| **테스트 항목** | 12개 주요 기능, 30개 이상의 시나리오 |
| **대상 페이지** | 메인 페이지, 검색 결과, 분석 결과 (3개 탭) |

## 3. 발견된 문제점

### 3.1严重 문제 (Critical - 즉시 수정 필요)

#### 问题 #1: 하드코딩된 한국어 텍스트로 인한 다국어 미지원

**영향 범위**: 모든 언어 (EN, ES에서严重)

**발생 위치 및 내용**:

| # | 화면 위치 | 하드코딩 텍스트 | 현재 언어 | 예상 동작 |
|---|---------|----------------|----------|----------|
| 1 |候选人 선택 UI | "동명이인이 있습니다. 해당하는 분을 선택해 주세요" | ALL | 언어별 번역 필요 |
| 2 |候选人 선택 UI 닫기 버튼 | "✕ 닫기" | ALL | 언어별 번역 필요 |
| 3 |候选人 검색 중 로딩 텍스트 | "Wikipedia에서 후보를 검색하는 중..." | ALL | 언어별 번역 필요 |
| 4 |구글 자동 수집 버튼 | "구글 자동 수집" | ALL | 언어별 번역 필요 |
| 5 |Missing MBTI Modal 제목 | "MBTI 정보가 없습니다" | ALL | 언어별 번역 필요 |
| 6 |Missing MBTI Modal 버튼 1 | "구글 AI로 자동 검색하기" | ALL | 언어별 번역 필요 |
| 7 |Missing MBTI Modal 버튼 2 | "직접 입력할래요" | ALL | 언어별 번역 필요 |
| 8 |Missing MBTI Modal 버튼 3 | "없이 순수 사주로만 분석하기" | ALL | 언어별 번역 필요 |

**영향받는 사용자 경험**:
- 영어/스페인어 사용자가 한국어 텍스트를 이해하지 못함
- 서비스의 전문성 저감
- 다국어 지원 선언과 실제 동작의 불일치

**수정 방안**:
- `frontend/src/App.tsx`의 `translations` 객체에 누락된 텍스트 추가
- 하드코딩된 모든 텍스트를 `t()` 함수로 교체

---

### 3.2 중요 문제 (High - 조속 수정 필요)

#### 问题 #2: 분석 결과 (Fortune Calendar) 언어 미반영

**现象**:
- 영어/스페인어 모드에서도 운세 캘린더의 모든 내용이 한국어로 표시됨
- 예시: "1월", "인생네컷", "럭키비키", "소확행" 등

**원인 분석**:
- Frontend: 이미 `lang` 파라미터를 API로 전송하고 있음 (line 328 확인)
- Backend: `saju_engine.py`에서 언어별 번역 생성 로직이 누락됨

**영향받는 탭**:
- 2026 God-saeng Calendar (월별 운세)
- Destiny Signal (분석 결과)

**수정 방안**:
- `backend/saju_engine.py`에 언어별 번역 생성 로직 추가
- 분석 결과 데이터에 번역된 텍스트 포함하여 반환

---

### 3.3 경미한 문제 (Medium - 개선 권장)

#### 问题 #3: Birth Date / MBTI 라벨의 t() 함수 미사용

**위치**: Idol 데이터 표시 영역
- "Birth Date" → t('birthDate') 사용 필요
- "MBTI-T/A" → t('mbti') 사용 필요

**영향**: 라벨만 해당하며 기능에는 영향 없음

## 7. 결론 및 권장사항

### 7.1 결론

UAT 테스트 결과, K-Destiny AI 서비스는 **다국어 지원 측면에서严重한 문제점**이 발견되었다. Frontend의 하드코딩된 한국어 텍스트와 Backend의 번역 로직 누락으로 인해 영어/스페인어 사용자의 서비스 이용에 직접적인 영향을 미치고 있다.

### 7.2 권장사항

1. **즉시 조치**: Phase 1 (Frontend 수정)을 즉시 진행하여 하드코딩 문제 해결
2. **긴급 조치**: Phase 2 (Backend 수정)를 병행하여 번역 로직 구현
3. **품질 보장**: 모든 수정 후 완전한 회귀 테스트 수행
4. **지속적 모니터링**: 다국어 테스트를 CI/CD 파이프라인에 통합 검토
