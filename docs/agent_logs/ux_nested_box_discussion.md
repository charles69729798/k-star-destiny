# 🧠 UX 전문가 토론 결과 보고서: 중첩 박스 구조 최적화

**토론 주제**: PC/모바일 하이브리드 서비스에서의 중첩 박스(Box-in-a-Box) 구조 및 여백/테두리 전략

---

## 👥 참여 전문가 의견 요약

### 1. Lead UX Architect (정보 설계 및 계층 구조)
> "현재의 '박스 안의 박스' 구조는 정보의 **'구획화'**에는 유리하지만, 모바일처럼 좁은 화면에서는 **'가둠 효과(Containment Effect)'**를 유발하여 시각적 답답함을 가중시킵니다."

- **핵심 진단**: 중첩된 테두리는 사용자 시선을 안쪽으로 가두어 실제 콘텐츠보다 화면이 더 작게 느껴지게 함.
- **의견**: 
    - **PC**: 정보가 많으므로 2단계 중첩까지는 허용하되, 배경색 차이(Shade)로 구분하고 선형 테두리는 최소화할 것.
    - **모바일**: 중첩을 1단계로 줄이거나, 테두리 대신 '여백(Whitespace)' 위주로 정보를 분리해야 함.

### 2. Mobile UX Specialist (모바일 최적화 및 접근성)
> "모바일에서 16px(1rem)씩 3번 중첩된 여백은 가로폭의 약 30%를 낭비합니다. **'엣지 투 엣지'** 디자인이 필수적입니다."

- **여백 폭 제안**:
    - **외부 테두리 - 내부 상자 사이**: 모바일에서는 **0~4px**로 최소화 (거의 붙여야 함).
    - **내부 상자 - 콘텐츠 사이**: 최소 **12~16px** 확보하여 텍스트 가독성 유지.
- **테두리 두께**: 1px 이상은 금지. 모바일에서는 선(Line)보다 면(Plane)의 대비를 추천.

### 3. Visual System Designer (디자인 시스템 및 심미성)
> "중첩된 박스마다 적용된 `rounded-2xl`과 같은 곡률이 서로 충돌하고 있습니다. **'곡률의 조화(Corner Radius Harmony)'**가 필요합니다."

- **디자인 가이드**:
    - 외부 박스의 곡률이 24px이라면, 내부 박스는 16px, 그 안은 8px 식으로 안으로 갈수록 곡률이 작아져야 수학적 안정감을 줌.
    - 테두리(Border) 두께는 동일하게 가져가되, 안쪽 상자의 테두리 색상을 더 연하게 처리하여 깊이감(Depth)을 조절할 것.

### 4. MZ Sentiment Analyst (MZ 감성 및 톤앤매너)
> "요즘 MZ 세대는 '답답한 상자'보다 **'공기감(Airy)'**이 느껴지는 레이아웃을 선호합니다. 테두리가 많으면 올드(Old)해 보입니다."

- **스타일 제안**:
    - 박스 형태는 유지하되 한쪽 면만 선을 쓰거나, 그림자(Drop Shadow)로만 경계를 나누는 방식 권장.
    - '나와 스타의 연결'이라는 컨셉에 맞게 경계가 너무 딱딱하지 않고 부드렵게 이어지는 느낌이 중요.

## 📊 취합된 최종 권장 안 (Consolidated Feedback)

| 항목 | 모바일 (Mobile) | PC (Desktop) | 비고 |
| :--- | :--- | :--- | :--- |
| **최외곽 여백** | 0~8px (최대한 확장) | 24~40px (여유로운 배치) | 모바일은 화면 활용 극대화 |
| **중첩 단계** | 최대 1단계 (Flat 지향) | 최대 2단계 (Tab/Section) | 중첩이 많을수록 인지 부하 증가 |
| **박스간 여백** | 8~12px (수직 간격) | 20~30px (수직/수평) | 시각적 그룹핑 용도 |
| **테두리 두께** | 1px 또는 제거 (면 분할) | 1px (Solid/Dashboard) | 선보다 면(BG Color) 추천 |
| **디자인 전략** | Edge-to-Edge 레이아웃 | 카드 기반 그리드 레이아웃 | 반응형 분기점(768px) 기준 |
