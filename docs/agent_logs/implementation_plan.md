# 아바타 로직 복구 및 프리미엄 리소스 구축 계획

프로젝트 초기 및 중기에 사용되었던 '성별 기반 4종 아바타 로테이션' 기능을 복구하고, 유실된 리소스를 최신 디자인 트렌드에 맞게 재생산합니다.

## 제안 배경
- 기존에 함께 만들었던 아바타 리소스가 폴더 구조 변경 과정에서 유실됨.
- 현재의 DiceBear 기본 아바타는 앱의 프리미엄 디자인(Glassmorphism, Neon)과 다소 이질감이 있음.
- 사용자 요청에 따라 성별당 4종의 고정 아바타를 번갈아 보여주는 로직으로 원복.

## 주요 변경 사항

### 1. 리소스 복구 및 생성 [New]
- **폴더 생성**: `frontend/public/avatars/`
- **리소스 생성**: `generate_image`를 활용하여 K-Pop 감성의 프리미엄 3D 아바타 8종 제작
  - `male_1.png` ~ `male_4.png`
  - `female_1.png` ~ `female_4.png`

### 2. Frontend 로직 수정 (App.tsx) [Modify]
- **이미지 서빙 우선순위 확립**:
  1. **특정 아이돌 매칭**: `public/custom-avatars/[name].png`가 존재할 경우 최우선 사용.
  2. **성별 로테이션**: 특정 이미지가 없을 경우, 기존처럼 `(name.length % 4) + 1` 공식을 활용해 `/avatars/[gender]_[n].png` 사용.
  3. **폴백**: 최후의 수단으로 DiceBear API 사용.

## 검증 계획
- `npm run dev` 실행 후 '인기 아이돌' 섹션에서 각 아이돌이 서로 다른 (하지만 성별에 맞는) 고정 아바타를 가지고 있는지 확인.
- 수동 검색 시에도 해당 로직이 정상 작동하는지 검증.
