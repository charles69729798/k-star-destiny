# 📱 모바일 가로폭 및 가독성 심층 아디트 보고서

사용자가 지적한 "답답한 가로폭" 문제를 해결하기 위해 UX 전문가 10인이 모여 분석한 결과입니다.

---

## 🧐 전문가 10인 모바일 가로폭 집중 분석

### 1. 📏 UI Architecture Expert
> "현재 이미지상으로 좌우 테두리(Stripe Background)가 전체 너비의 약 30% 이상을 차지하고 있습니다. 모바일에서는 1px이 아쉬운 상황인데, 이 디자인은 데스크탑 중심의 고정폭 레이아웃을 모바일에서 그대로 축소한 것처럼 보입니다. `max-width: 100%`와 `padding: 1rem` 수준으로 여백을 대폭 줄여야 합니다."

### 2. 👁️ Readability Coordinator
> "한 줄에 10자 내외의 텍스트는 시각적으로 '파편화'를 유도합니다. 문장이 너무 자주 끊기면 사용자는 글을 읽는 것이 아니라 '단어 찾기'를 하게 됩니다. 최소 한 줄에 18~22자 정도는 확보되어야 자연스러운 시선 이동이 가능합니다."

### 3. 🎨 Minimalist Designer
> "배경의 세로 스트라이프가 텍스트 영역을 가두는 '감옥'처럼 느껴집니다. 모바일에서는 배경 패턴을 과감히 제거하거나 투명도를 높여 시각적 확장성을 부여해야 합니다. 경계선을 없애고 '엣지 투 엣지' 디자인을 지향하세요."

### 4. ⚡ MZ Trend Hunter
> "MZ 세대는 시원시원한 '풀 스크린' 경험을 선호합니다. 현재의 UI는 2000년대 피처폰 웹 페이지 같은 느낌을 줍니다. 테두리를 과감히 버리고 텍스트가 화면 끝까지 닿을 듯 말 듯한 '아슬아슬한 여백'이 더 힙(Hip)합니다."

### 5. 📱 Android UX Specialist
> "안드로이드의 시스템 내비게이션 바와 브라우저 툴바까지 고려하면 실제 가용 영역은 더 좁습니다. 카드 컨테이너의 `mx-auto`와 `max-width` 설정을 모바일 전용 미디어 쿼리(`@media (max-width: 640px)`)를 통해 유연하게 풀어주어야 합니다."

### 6. 🛠️ Frontend Performance Engineer
> "단순히 여백을 줄이는 것뿐만 아니라, 폰트 크기를 0.1~0.2rem 정도 줄여서 상대적으로 더 많은 글자가 들어가게 조절하는 것도 전략입니다. 하지만 가장 큰 문제는 컨테이너에 걸린 `px-10` 같은 과도한 패딩값으로 보입니다."

### 7. 🎯 Conversion Rate Optimizer
> "글이 답답하면 사용자는 끝까지 읽지 않고 이탈합니다. 특히 사주/운세 서비스는 텍스트가 핵심 상품인데, 그 상품을 보관하는 상자가 너무 좁아 상품 가치를 훼손하고 있습니다. 'Full-Width' 모드로 전환이 시급합니다."

### 8. ✨ Interaction Designer
> "좌우 테두리가 두꺼우면 사용자는 화면을 스와이프할 때 좌우 영역을 '데드 존(Dead Zone)'으로 인식합니다. 텍스트가 더 넓게 배치되면 터치 조작 범위도 넓어져 사용성이 개선됩니다."

### 9. 🎨 Color & Contrast Expert
> "배경 스트라이프와 텍스트 카드의 색상 대비가 강해 경계가 너무 뚜렷합니다. 이 경계가 벽처럼 느껴지는 것입니다. 배경색을 단일화하거나 그라데이션을 사용해 공간감을 넓혀야 합니다."

### 10. 🏆 Senior Product Designer
> "결론적으로, 모바일에서는 'Main Container'의 좌우 여백을 기존의 1/3 수준으로 줄이고, 배경 스트라이프 디자인은 모바일에서만 숨기거나(display: none) 훨씬 좁게 배치하여 텍스트의 '호흡'을 열어주어야 합니다."

---

## 🛠 긴급 개선 권고안 (Action Plan)

1.  **배경 스트라이프 최적화**: 모바일 해상도에서 양옆의 어두운 배경 영역을 제거하거나 5% 이하로 축소.
2.  **컨테이너 너비 확장**: 메인 콘텐츠 카드의 `max-width`를 모바일에서 `95vw` 이상으로 상향.
3.  **패딩 리셋**: 모바일 전용 스타일로 좌우 패딩을 `px-4`(약 16px) 이하로 재설정.
4.  **폰트 밸런싱**: 폭 확보 후 텍스트 정렬을 '양쪽 정렬' 또는 '좌측 정렬'로 최적화하여 가독성 극대화.
