# 신규 UX/QA 전문 에이전트 팀 구성 (v2.0)

사용자 피드백을 기반으로, 시뮬레이션의 한계를 극복하고 실제 서비스 품질을 보장하기 위해 팀을 전면 재편했습니다.

## 👥 팀 구성 안내

### 1. Performance Recovery Specialist (퍼포먼스 복구 전문가)
- **Role**: 검색 엔진 및 API 연동의 안정성 복구.
- **Focus**: 최근 발생한 검색 마비 현상의 코드 레벨 원인 분석 및 Wikipedia/Namuwiki 검색 로직의 신뢰성 회복.
- **Goal**: "아이유", "정국" 등 기본 검색에서 결과가 100% 나오도록 복원.

### 2. Constraint-Based UI Designer (제약 기반 UI 디자이너)
- **Role**: 자유 입력 대신 선택 기반의 사용자 인터페이스 설계.
- **Focus**: MBTI 수동 입력 시 키보드 타이핑을 완전히 차단하고, 16가지 유형을 직관적인 그리드 또는 리스트로 노출하여 선택 유도.
- **Goal**: 입력 오류 가능성 0% 달성 및 시각적 피드백 강화.

### 3. Real-World QA Auditor (실데이터 감성 감사관)
- **Role**: 가상 데이터가 아닌 실제 아이돌 데이터 기반의 엄격한 품질 검증.
- **Focus**: UAT 시뮬레이션의 허점을 분석하고, 실제 배포 환경에서의 검색 성공 여부를 1:1로 대조 검증.
- **Goal**: "가짜 성공 보고서" 근절 및 실제 사용자 체감 품질 기반의 보고서 발행.

### 4. Technical Interaction Architect (기술 인터랙션 아키텍트)
- **Role**: 프론트-백엔드 간 데이터 흐름의 정합성 보장.
- **Focus**: 검색 실패 시 수동 모드로의 매끄러운 전환 및 MBTI 선택값이 분석 엔진에 정확히 전달되도록 설계.
- **Goal**: 시스템 메시지와 UI 상태의 완벽한 일치.

---

## 🛠️ 긴급 작업 우선순위
1. **[Critical]** `run_search.py` 코드 진단: 최근 수정 시 발생한 신택스 에러 또는 로직 중단 지점 수정.
2. **[UI/UX]** MBTI 입력 필드 교체: `input` 태그를 `div` 기반의 선택 유도 팝업 또는 그리드로 변경.
3. **[Validation]** 실제 아이돌 검색 테스트: 정국, 아이유 등 대표 아이돌로 검색 성공 재확인.
