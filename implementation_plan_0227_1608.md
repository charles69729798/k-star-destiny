# 실행 계획서: 에이전트 UI 블랙화면 복구 및 환경 최적화 (0227_1608)

이 계획서는 사용자가 겪고 있는 에이전트 화면 블랙아웃 현상을 해결하기 위해 UI 부하를 유발하는 불필요한 파일들을 정리하고, 에이전트 로딩 성능을 개선하는 것을 목표로 합니다.

## 제안된 변경 사항

### 1. UI 부하 원인 제거 및 복구 안내
- **원인 분석**: 1) 이전 대화에 쌓인 850개 이상의 아티팩트/로그 데이터 파싱 부하, 2) 프로젝트 루트에 산재한 20여 개의 임시 파일(`temp_*`) 스캔 지연.
- **복구 안내**: 사용자가 무한 대기하지 않도록 UI 새로고침(F5) 또는 앱 재시작을 권고합니다.

### 2. 프로젝트 루트 디렉토리 최적화
- **임시 파일 정리**: `temp_App_*.tsx`, `temp_index_*.css` 등 작업 부산물들을 `backup_temp_files` 폴더로 일괄 이동하여 에이전트의 파일 인덱싱 부하를 획기적으로 줄입니다.
- **예약어 파일 재점검**: `nul` 파일이 완전히 제거되었는지 다시 한번 확인합니다.

### 3. 향후 재발 방지 지침 (instructions.md 업데이트)
- "프로젝트 루트 디렉토리에는 핵심 소스 파일과 공식 리포트(`_MMDD_HHMM.md`)만 유지하며, 임시 파일은 정기적으로 정리한다"는 원칙을 명시합니다.

---

## 검증 계획

### 자동화 테스트
- `ls temp_*` 명령을 실행하여 프로젝트 루트에 임시 파일이 남아있는지 확인합니다.
- `dir /a nul` 명령으로 예약어 파일의 부재를 최종 확인합니다.

### 수동 검증 (사용자 협조 필요)
1. 사용자가 UI를 새로고침한 후, 대화창이 블랙화면 없이 정상적으로 로딩되는지 확인합니다.
2. 에이전트 응답 속도가 이전보다 개선되었는지 체감 성능을 확인합니다.
