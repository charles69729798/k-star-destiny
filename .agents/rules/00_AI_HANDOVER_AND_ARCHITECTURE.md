# 🚨 AI 요원 (Agent) 작동 시작 시 필수 숙지 사항

당신(AI)은 K-Star Destiny 프로젝트의 새 대화 세션에 막 투입되었습니다. 작업을 시작하기 전에 아래의 아키텍처와 인수인계 상태를 반드시 파악하십시오.

## 1. 아키텍처 및 서버 배포 환경 요약
- **단일 서버 & 실시간 배포 (Hot-Reloading):** 이 Windows PC 자체가 전 세계로 서비스 중인 라이브 서버입니다. GitHub Push-Pull을 통한 전통적인 서버 배포를 시도하지 마십시오.
- **Docker Volume Mount:** `.backend` 파일(Python)이나 `.frontend` 파일(React)을 수정하고 저장하면, 연결된 도커 컨테이너가 스스로 감지하여 인터넷 서비스(`destiny-signal.duckdns.org`)에 1초 만에 즉시 반영됩니다.
- **구조:** 
  - `설정/서빙`: Nginx (`sajuapp-gateway`)가 80/443 포트를 잡고 있습니다. (인증서는 `./deploy/certbot/` 아래 존재)
  - `프론트엔드`: Vite/React 기반. (`sajuapp-frontend`)
  - `백엔드`: FastAPI/Python 기반. 알고리즘 연산 수행. (`sajuapp-backend`)

## 2. 세션 인수인계(Handover) 프로토콜
이전 대화창의 AI 요원이 메모리 한계나 오류로 작업을 중단했을 수 있습니다. 당신은 다음 단계를 밟아야 합니다.

1. **상태 확인:** 즉시 프로젝트 최상단 폴더에 있는 `CURRENT_STATE.md` 파일을 읽으십시오(`view_file` 도구 사용).
2. **이전 작업 파악:** 그 파일 안에는 '바로 직전까지 무슨 에러를 고치고 있었는지', '어느 소스 코드를 만지다 끊겼는지'가 상세히 적혀 있습니다.
3. **작업 이어서 재개:** 사용자가 번거롭게 이전 상황을 처음부터 다시 설명하게 하지 마십시오. 스캔한 `CURRENT_STATE.md`의 내용을 바탕으로 "아~ 이전 창에서 프론트엔드 어느 부분을 고치고 계셨군요! 제가 이어서 점검하겠습니다"라고 자연스럽게 대화와 작업을 이어가십시오.

## 3. 대화 세션 종료 전 AI의 의무
- 대화창이 꽉 차서 더 이상 진행이 어렵거나, 오늘 작업이 끝나는 시점에 도달하면, 당신(AI)은 반드시 최상단의 `CURRENT_STATE.md` 파일에 **오늘 수정한 파일 목록, 남은 버그, 다음번에 해야 할 일**을 요약해서 업데이트(저장) 해두어야 합니다.
