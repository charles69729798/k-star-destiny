# K-Destiny Project Local Rules
---
description: 에이전트 반복 오류 방지를 위한 프로젝트 로컬 환경 규칙입니다.
---

## 1. 프론트엔드 API 연결(Port) 에러 방지 규칙
- 로컬 환경(npm run dev)에서 프론트엔드와 백엔드(.env)가 통신할 때, **백엔드 서버가 8000번 포트로 직접 응답하지 않고 Nginx(80 포트) 도커 컨테이너를 경유해야만 CORS를 통과 및 정확한 데이터를 가져올 수 있습니다.**
- 따라서 프론트엔드를 구동하기 전, 반드시 `frontend/.env.local` 파일 내에 `VITE_API_URL=http://localhost/api` 가 설정되어 있는지 확인하거나 강제로 생성해야 합니다.
- (관련 워크플로우: `/start` 실행 시 이 동작이 기본 적용되도록 조치됨)

## 2. 모바일 UI 등 렌더링 검토 규칙
- 줄바꿈이나 자간 파괴 등 텍스트 수직 렌더링 문제 해결을 위해, tailwindcss 설정 시 좁은 화면 디바이스(`sm:`, `md:` 중단점 미만)를 고려한 반응형 css 클래스를 중시합니다.
