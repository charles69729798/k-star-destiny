# K-Destiny UAT 핸드오버 가이드 (타 AI/협업자용)

이 문서는 다른 AI 에이전트나 테스트 협업자가 본 프로젝트의 UAT(사용자 수용 테스트)를 즉시 수행할 수 있도록 서버 구동 및 테스트 환경 정보를 제공합니다.

---

## 🏗️ 1. 서버 구동 방법 (Server Setup)
테스트를 시작하기 전, 백엔드와 프론트엔드 서버가 모두 실행 중이어야 합니다.

### A. 백엔드 서버 (FastAPI)
- **위치**: `/backend`
- **명령어**: `python main.py`
- **정상 상태**: `http://localhost:8000`에서 API 응답 가능 상태

### B. 프론트엔드 서버 (Vite)
- **위치**: `/frontend`
- **명령어**: `npm run dev -- --host`
- **정상 상태**: `http://localhost:5173` 접속 가능 상태

---

## 🔗 2. 접속 및 테스트 정보
- **테스트 URL**: [http://localhost:5173](http://localhost:5173)
- **주요 검증 포인트**:
    1. **MBTI Picker**: MBTI 입력창 클릭 시 16가지 유형 선택 팝업이 즉시 뜨는지 확인.
    2. **지인 매칭 모드**: 상단 탭 전환 후 지인 정보 입력 및 MBTI 선택 기능 확인.
    3. **실시간 반영**: 코드 수정 후 브라우저에 즉시 반영되는지 확인 (HMR 동작 여부).

---

## ⚠️ 3. 필독 주의 사항 (Troubleshooting)
테스트 중 화면이 변하지 않거나 연결이 안 될 경우 다음 조치를 AI에게 지시하십시오.

1. **강력한 새로고침**: 브라우저 캐시로 인해 구버전이 노출될 수 있으므로 반드시 **Ctrl + Shift + R**을 수행해야 함.
2. **좀비 프로세스 청소**: 5173 또는 8000 포트가 꼬였을 경우 `taskkill /F /IM node.exe /T` 등을 통해 깨끗이 비우고 재시작할 것.
3. **서버 캐시 삭제**: 수정 사항이 안 보일 경우 `frontend/node_modules/.vite` 폴더를 삭제하고 서버를 재시작할 것.
4. **Vite 설정**: 파일 변경 감지가 안 될 경우 `vite.config.ts`의 `usePolling: true` 설정을 확인할 것 (현재 적용 중).

---

## 📋 4. 권장 UAT 시나리오
1. **스타 모드**: 아이유(IU) 검색 -> MBTI Picker로 유형 선택 -> 분석 실행.
2. **지인 모드**: 이름 입력 -> 성별 선택 -> MBTI Picker 연동 확인 -> 분석 실행.
3. **반응형 테스트**: 모바일 뷰(375px)에서 레이아웃 깨짐 현상 및 터치 용이성 확인.
