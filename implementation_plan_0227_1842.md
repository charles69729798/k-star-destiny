# 실행 계획서: 에이전트 인덱싱 장애 해결 및 시스템 아키텍처 정규화 (0227_1842)

대화 데이터 정리 후에도 로딩이 지속되는 원인을 **".claudecode 폴더 내의 파일명 손상"** 및 **"프로젝트 내 거대 로그 파일에 의한 인덱싱 지연"**으로 규정하고 이를 최종 정리합니다.

## 제안된 변경 사항

### 1. 손상된 .claudecode 스킬 파일 삭제
- **현상**: `SKILL.mdL.md`, `SKILL.mdystem\SKILL.md` 등 비정상적인 경로/파일명이 존재하여 에이전트의 기능 로딩 시 예외를 유발할 가능성 농후.
- **조치**: 해당 비정상 파일들을 삭제하여 상태 관리의 정합성을 확보합니다.

### 2. 거대 로그 파일 아카이빙 (10MB 이상)
- **현상**: `docs/agent_logs` 내에 수십 MB에 달하는 로그 파일들이 존재하여 에이전트가 프로젝트 전체를 스캔할 때 병목 현상 발생.
- **조치**: 10MB 이상의 모든 로그 및 결과 파일을 `backup_large_files` 폴더로 격리합니다.

### 3. 윈도우 예약어 파일 재검증
- 다시 한번 `nul`, `con` 등의 예약어 파일이 숨겨진 경로에 존재하는지 전수 조사하여 삭제합니다.

## 검증 계획

### 자동화 테스트
- `Get-ChildItem`을 통해 10MB 이상의 파일이 루트 및 주요 작업 경로에서 사라졌는지 확인합니다.
- `.claudecode` 내의 `SKILL.md` 파일들이 정규화된 이름으로만 존재하는지 확인합니다.

### 수동 검증 (사용자)
- Antigravity 앱을 완전히 종료 후 재시작하여 "Agent Loading" 메시지 없이 즉각적으로 대화 리스트가 뜨는지 확인합니다.
